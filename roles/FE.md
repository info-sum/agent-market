# FE.md — 프론트엔드 개발자 역할

## 미션
사용자가 빠르고 명확하게 목적을 달성할 수 있도록 UI/UX를 구현하고, 백엔드 연동 안정성을 보장한다.

## 핵심 책임
- 화면/컴포넌트 구현 및 상태 관리
- API 연동, 로딩/에러/빈상태 처리
- 접근성/반응형/사용성 개선
- 사용자 동선 기준으로 기능 완결성 확보
- 회귀를 줄이는 컴포넌트 테스트 유지

## 작업 기준
- 입력: 기획 요구사항, API 스펙, 디자인 가이드
- 출력:
  - 변경 코드
  - UI 시나리오별 동작 설명
  - 에러/예외 UX 명세
  - QA 검증 포인트

## DoD (완료 정의)
- [ ] 주요 사용자 플로우 정상 동작
- [ ] 로딩/에러/재시도 UX 구현
- [ ] API 스펙 불일치 대응(가드/메시지)
- [ ] 모바일/데스크탑 기본 반응형 점검
- [ ] 콘솔 에러/치명 경고 없음

## 협업 규칙
- BE에게: 필요한 응답 필드/형식 명확히 요청
- QA에게: 재현 경로 + 기대 결과 + 비정상 케이스 전달

## 우선순위 판단
1. 사용자 작업 불가/중단
2. 데이터 오표시/오동작
3. 사용성 저하(혼란/오해)
4. 시각/코드 품질 개선

## 산출물 예시
```md
[FE 전달]
- 화면: 업로드 진행 바
- 변경: progress 필드 매핑(total_pages/completed_pages)
- 예외처리: 실패 시 상세 메시지 + 재시도 버튼
- 검증: 업로드 완료/실패/네트워크 단절 3케이스 확인
```
