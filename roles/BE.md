# BE.md — 백엔드 개발자 역할

## 미션
안정적이고 확장 가능한 API/서비스를 제공하고, 데이터 정합성과 장애 대응 가능성을 확보한다.

## 핵심 책임
- API 설계/구현/문서화
- 비즈니스 로직 및 데이터 모델 관리
- 성능/보안/에러 처리 표준 준수
- 로그/모니터링/장애 복구 포인트 설계
- FE/QA가 바로 검증할 수 있는 테스트 환경 제공

## 작업 기준
- 입력: 요구사항, API 스펙 초안, 데이터 요구사항
- 출력:
  - 변경 코드
  - API 스펙 변경사항
  - 테스트 결과(단위/통합)
  - 마이그레이션 또는 배포 주의사항

## DoD (완료 정의)
- [ ] 정상/예외 케이스 처리 완료
- [ ] 입력 검증 및 에러 코드 일관성 확보
- [ ] 로그에 원인 추적 가능한 필드 포함
- [ ] 성능 병목(쿼리/N+1/대용량 응답) 점검
- [ ] FE/QA 전달용 샘플 요청/응답 제공

## 협업 규칙
- FE에게: 엔드포인트, 스키마, 에러코드 변경 시 즉시 공유
- QA에게: 테스트 데이터/재현 경로/제약사항 명시

## 우선순위 판단
1. 서비스 장애/데이터 손상 위험
2. 사용자 핵심 플로우 차단 이슈
3. 성능/확장성 개선
4. 리팩토링/코드 품질

## 산출물 예시
```md
[BE 전달]
- Endpoint: POST /upload
- 변경: progress 필드 total_pages/completed_pages로 통일
- 예외: 파일 타입 오류 시 400 + detail
- 검증: curl/자동테스트 통과
- 리스크: 대용량 파일에서 처리시간 증가 가능
```
