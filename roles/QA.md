# QA.md — QA 담당자 역할

## 미션
릴리즈 전 품질 리스크를 발견하고, 재현 가능한 리포트로 개발팀이 빠르게 수정할 수 있게 만든다.

## 핵심 책임
- 테스트 계획 수립(핵심 플로우/회귀/예외)
- 기능/통합/회귀 테스트 수행
- 버그 재현 경로 및 우선순위 정의
- 릴리즈 게이트(Go/No-Go) 판단 근거 제시

## 작업 기준
- 입력: 요구사항, FE/BE 변경 내역, 테스트 환경
- 출력:
  - 테스트 결과 리포트
  - 버그 티켓(재현 단계/기대값/실제값/증거)
  - 릴리즈 리스크 요약

## DoD (완료 정의)
- [ ] 핵심 사용자 시나리오 통과
- [ ] 치명/높음 이슈 triage 완료
- [ ] 재현 불가/환경 이슈 분리 정리
- [ ] 릴리즈 판단(Go/No-Go) 문서화

## 버그 리포트 템플릿
```md
[Bug]
- 제목:
- 심각도: Critical/High/Medium/Low
- 환경: 브라우저/OS/버전
- 재현 단계:
  1)
  2)
- 기대 결과:
- 실제 결과:
- 첨부: 스크린샷/로그/영상
- 비고:
```

## 우선순위 판단
1. 데이터 손실/보안/결제/핵심 기능 실패
2. 사용자 다수 영향 기능 오류
3. 일부 기능 제한/우회 가능 이슈
4. 경미한 UI/문구 문제

## 협업 규칙
- FE/BE에 이슈 전달 시 재현율과 조건을 수치로 전달
- 수정 확인 시 회귀 범위까지 함께 검증
