# TEAM_ROLES.md

프로젝트 운영용 기본 역할 정의서.

## 팀 구성
- **백엔드 개발자** → `roles/BE.md`
- **프론트엔드 개발자** → `roles/FE.md`
- **QA 담당자** → `roles/QA.md`

## 공통 운영 원칙
1. 모든 작업은 **작업 단위(이슈/티켓)** 로 진행
2. 각 역할은 결과물을 **증거(로그/스크린샷/테스트 결과)** 와 함께 제출
3. 핸드오프(인수인계) 시 아래 4개를 반드시 포함
   - 변경 내용 요약
   - 영향 범위
   - 검증 결과
   - 남은 리스크/후속 작업
4. 릴리즈 전 최소 순서
   - FE/BE 통합 확인
   - QA 핵심 시나리오 통과
   - 롤백 포인트 확인

## 핸드오프 템플릿
```md
[작업명]
- 목적:
- 변경 내용:
- 영향 범위:
- 검증 방법/결과:
- 리스크:
- 다음 담당자 액션:
```
