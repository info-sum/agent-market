# 에이전트 모델화/스킬 마켓 플랫폼 기획서 (v1)

## 1) 한 줄 정의
**각 사용자의 OpenClaw를 토큰으로 안전하게 연결하고, 외형(캐릭터/페르소나) + 능력(Skill/MD 파일)을 사고팔아 에이전트를 성장시키는 마켓 플랫폼**

---

## 2) 문제 정의
현재 OpenClaw 사용자들은:
- 세팅/운영 경험이 분산되어 있고 재사용이 어렵고,
- “잘 일하는 에이전트”를 만드는 노하우(MD, 스킬, 운영 규칙)가 개인에게 묶여 있으며,
- 외형/캐릭터/톤 커스터마이징 자산이 표준화되지 않아 공유·거래가 어려움.

즉, **에이전트 품질을 높이는 자산은 있는데 유통 구조가 없음**.

---

## 3) 제품 비전
1. **연동 허브**: 내 OpenClaw를 웹에서 한 번에 관리
2. **커스터마이징 스튜디오**: 에이전트 외형/페르소나 “옷입히기”
3. **스킬/MD 마켓**: 검증된 생산성 자산 거래
4. **성과 기반 생태계**: 구매 후 성능 개선이 보이는 구조

---

## 4) 핵심 사용자
- **Buyer (일반 사용자/팀 리더)**: 내 에이전트를 더 똑똑하게 만들고 싶은 사람
- **Creator (개발자/디자이너/프롬프트 설계자)**: 스킬, MD 패키지, 캐릭터 자산 판매자
- **Operator (관리자/심사자)**: 품질·보안·신뢰 정책 운영

---

## 5) 핵심 가치 제안 (Value Proposition)
### Buyer에게
- 설치형 템플릿으로 빠르게 성능 개선
- 카테고리별(개발/디자인/PM/마케팅) 즉시 도입
- 구매 전 미리보기 + 구매 후 성과지표 확인

### Creator에게
- 디지털 자산 수익화 채널
- 버전 관리/업데이트 배포/리뷰 시스템
- 평판 기반 노출 알고리즘

---

## 6) 제품 구조
## 6.1 계정/연동
- OAuth 또는 API Token 방식으로 사용자 OpenClaw 연결
- 권한 범위(Scope) 분리: 읽기/쓰기/설치/실행
- 토큰 암호화 저장 + 주기적 재발급(rotate) 지원

## 6.2 에이전트 “옷입히기” (Customization)
- 캐릭터 프리셋(이름/톤/역할/응답 스타일)
- 아바타/테마/UI 스킨
- 시스템 프롬프트 템플릿(안전 가드레일 포함)

## 6.3 스킬/MD 마켓
- 상품 유형
  - Skill Pack (기능 스크립트+설명+설치 가이드)
  - MD Pack (운영 규칙, 역할 가이드, 프롬프트 정책)
  - Persona Pack (캐릭터+톤+템플릿)
- 가격 정책: 무료/유료/구독
- 라이선스: 개인/팀/상업용

## 6.4 설치/적용 플로우
1) 마켓 탐색
2) 미리보기/리뷰 확인
3) 결제
4) 내 OpenClaw에 원클릭 적용
5) 롤백/버전 전환

---

## 7) 신뢰·보안 정책 (핵심)
- 악성 스킬 업로드 방지(정적 검사 + 수동 심사)
- 민감 권한 요청 시 명시적 동의
- 샌드박스 실행/권한 격리
- 설치 전 “변경될 파일/영향 범위” 고지
- 신고/환불/분쟁 처리 정책

> 이 플랫폼의 성패는 기능보다 **신뢰 체계**에 달려 있음.

---

## 8) 비즈니스 모델
- 거래 수수료 (예: 10~20%)
- Creator Pro 구독 (고급 통계/홍보 슬롯)
- Team 플랜 (팀 라이선스/권한관리/감사 로그)
- 추천/큐레이션 영역 광고 슬롯(후순위)

---

## 9) MVP 범위 (첫 출시)
### 포함
- OpenClaw 토큰 연동 (기본 스코프)
- 상품 등록/리스트/상세/구매
- MD Pack 설치(가장 단순한 단위부터)
- 기본 리뷰/평점
- 관리자 승인 워크플로

### 제외 (2차)
- 자동 성능 벤치마크 리포트
- 실시간 공동편집
- 고급 추천 알고리즘
- 다국가 결제/세금 자동화

---

## 10) 로드맵
### Phase 1 (0~6주): Foundation
- 토큰 연동, 사용자/크리에이터 구분, 상품 스키마 설계
- MD Pack 업로드/검수/설치 최소 기능

### Phase 2 (7~12주): Market Launch
- 결제/정산, 리뷰, 버전업데이트, 환불 정책
- 카테고리 오픈(개발/디자인/PM)

### Phase 3 (13~20주): Growth
- 추천/랭킹, 성과지표 대시보드, 팀 플랜
- Creator 툴(AB 패키지, 가격 실험)

---

## 11) KPI
- 연동 계정 수
- 상품 등록 수 / 승인율
- 구매 전환율
- 재구매율(30일)
- 설치 후 유지율(7/30일)
- 환불률/신고율

---

## 12) 리스크 & 대응
1. **품질 편차** → 검수 레벨/큐레이션/배지 시스템
2. **보안 이슈** → 권한 최소화 + 샌드박스 + 감사지표
3. **초기 공급 부족** → 핵심 크리에이터 시드 프로그램 운영
4. **효용 불명확** → 설치 전후 성과 비교 리포트 제공

---

## 13) 운영 조직 제안
- PM 1, Backend 1~2, Frontend 1, Security/Review 1, Design 1 (파트타임 가능)
- 초기에는 “검수 품질” 인력 우선 배치
